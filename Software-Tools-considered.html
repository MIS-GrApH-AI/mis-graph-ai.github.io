Software tools for the Human-Machine Interface

Etienne Saliez, version 2021-09-01 /

  • Introduction:
    • Adaptation and extension of existing software tools to be adapted in function of the requirement of the GrApH-AI project in healthcare context.
    • As far as possible the essential functionalities should be Open Source. Availability of Assisted Intelligence will be particularly useful in developing regions where there are very few doctors.
    • Collaborations in the Neo4j community. https://neo4j.com/developer/contributing-code/ .
    • Introduction to graph visualization tools: https://neo4j.com/developer/tools-graph-visualization/ ,
  • Neo4j browser:
    • Provide basic way to view a graph and to navigate. Accept cypher statements which is useful for debugging.
    • The " :style " option provide a way to convert systematically logical information as nodes and relations types into a visual style.
  • Vis-network community:
    • Exellent libraries covering most of our requirements.
    • https://visjs.org/ , https://github.com/visjs , examples: https://visjs.github.io/vis-network/examples/ ,
    • An Open Source fork of vis.js initially developed but no more maintained by the Almende company, https://almende.github.io/vis/ .
  • Neo4j Arrows:
    • Excellent example of interface for:
      • The creation of new nodes and relationships.
      • The maintenance of the attributes of node and relationships.
    • But intended only for discussions about small models.:
      • Do not accept multiple instances of the same type.
      • Can save a small model but cannot read nor write in an existing large graph data base.
    • https://neo4j.com/labs/arrows/ .
  • Neovis:
    • Provide easy tools for the conversion of logical attributes into visual aspects, as in vis-network. An official Neo4j library.
    • https://medium.com/neo4j/graph-visualization-with-neo4j-using-neovis-js-a2ecaaa7c379 ,created by William Lyon.
    • Arrows and neovis should be integrated ?
  • Popoto:
    • Interesting graph interface.
    • http://www.popotojs.com/ , https://github.com/Nhogs/popoto-examples , created by Frédéric Ciminera.
  • Bloom:
    • Intended for the analyse of large Neo4j databases by non IT users. Question can be formulated in a much easier way than "cypher".
    • https://neo4j.com/product/bloom/ . not free Open Source.
